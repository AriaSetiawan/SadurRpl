1. class diagram menggambarkan hubungan antar class
  2. class diagram terdiri dari nama class, atribut, method
  3. seteotype -> menentukan/menandakan attribute yang menjadi primary key maupun foreign key
  4. Multiplity -> menentukan tingkat hubungan ? kardinalitas antar class melalui asociation(garis hubungan atar class)
  5. 0..1 => dapat tidak ada(kosong) sampai dengan 1
  6. 1 => hanya 1 dan pasti hanya 1
  7. 0..* => dapat tidak ada (kosong) sampai dengan tak terhingga (banyak)
  8. * => dapat banyak (tak terhingga)
  9. 1..* => sedikitnya 1, bisa lebih dari 1
