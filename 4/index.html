Hasil Sanduran

1. Membuat file readme.md untuk perhitungan nilai kehadiran
2. file index.html untuk nilai tugas (sanduran)
3. usecase diagram dan deskripsi usecase
4. usecase diagram untuk mengambarkan hubungan apa yang bisa dilakukan aktor
5. usecase diagram tidak menjelaskan secara detail tentang penggunaan usecase, namun hanya memberi gambaran singkat hubungan antara usecase, aktor dan sistem
6. include => disetakan (panah ke arah usecase inti/induk)/pengembangan
7. Extend => pengembangan/berkelanjutan (panah ke arah usecase yang diperlukan)
8. Setiap usecase akan menjadi 1 rancangan layar saling berhubungan
9. Deskripsi usecase dan skenario usecase merupakan satu paket yang dibuat sebagai tabel Deskripsi Usecase
